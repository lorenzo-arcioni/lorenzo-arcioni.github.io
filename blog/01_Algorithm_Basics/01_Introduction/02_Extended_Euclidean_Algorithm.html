Extended Euclidean Algorithm

The Extended Euclidean Algorithm is an extension of the Euclidean Algorithm that not only computes the greatest common divisor (GCD) of two integers \(A\) and \(B\), but also finds the coefficients of Bézout's identity, which are the integers \(x\) and \(y\) such that \(Ax + By = \text{GCD}(A, B)\).


Algorithm Description

The Extended Euclidean Algorithm can be described using the following steps:


  1. Start with two positive integers \(A\) and \(B\), where \(A \geq B\).
  2. Perform the Euclidean Algorithm steps to find the GCD of \(A\) and \(B\).
  3. Simultaneously, compute the coefficients \(x\) and \(y\) such that \(Ax + By = \text{GCD}(A, B)\).
  4. Update \(x\) and \(y\) using the recursive relationships derived from the Euclidean Algorithm steps.

Mathematical Representation

The Extended Euclidean Algorithm can be mathematically represented as:


\[ \begin{aligned} &\text{Given } A \text{ and } B, \text{ with } A \geq B: \\ &\text{1. If } B = 0, \text{ then } \text{GCD}(A, B) = A \text{ and } x = 1, y = 0. \\ &\text{2. Otherwise, perform the Euclidean Algorithm:} \\ &\hspace{20px} A = BQ + R \text{, where } R = A \mod B \\ &\hspace{20px} \text{GCD}(A, B) = \text{GCD}(B, R) \\ &\hspace{20px} \text{Recursively find coefficients } x_1 \text{ and } y_1 \text{ such that } Bx_1 + Ry_1 = \text{GCD}(B, R) \\ &\hspace{20px} \text{Set } x = y_1 \text{ and } y = x_1 - Qy_1 \\ &\text{3. Return } \text{GCD}(A, B), x, \text{ and } y. \end{aligned} \]

Example

Let's find the GCD and Bézout coefficients of \(A = 252\) and \(B = 105\) using the Extended Euclidean Algorithm:


  1. Start with \(A = 252\) and \(B = 105\).
  2. Compute \(252 \mod 105 = 42\). \(252 = 105 \times 2 + 42\).
  3. Compute \(105 \mod 42 = 21\). \(105 = 42 \times 2 + 21\).
  4. Compute \(42 \mod 21 = 0\). \(42 = 21 \times 2 + 0\).
  5. Back-substitute to find \(x\) and \(y\):
    \(21 = 105 - 42 \times 2\)
    \(42 = 252 - 105 \times 2\)
    So, \(21 = 105 - 2(252 - 105 \times 2) = 105 \times 5 - 252 \times 2\).
    Thus, \(x = -2\) and \(y = 5\).

Hence, the GCD of 252 and 105 is 21, with Bézout coefficients \(x = -2\) and \(y = 5\), such that \(252(-2) + 105(5) = 21\).


Understanding the Extended Euclidean Algorithm

The Extended Euclidean Algorithm utilizes the following key properties:


  • It extends the Euclidean Algorithm by tracking the coefficients \(x\) and \(y\) in Bézout's identity.
  • The algorithm maintains the relationship \(Ax + By = \text{GCD}(A, B)\) throughout its steps.
  • It allows the computation of modular inverses, crucial in many cryptographic algorithms, including RSA.

Proofs

We can demonstrate the correctness of the algorithm through proofs:


  • Proof of Bézout's Identity: For integers \(A\) and \(B\), with GCD \(d\), there exist integers \(x\) and \(y\) such that \(Ax + By = d\). This is shown by the iterative process of back-substitution from the Euclidean Algorithm steps. \(\square\)

  • Proof that the Algorithm Produces Correct Coefficients:

    By induction on the steps of the Euclidean Algorithm:


    1. Base Case: If \(B = 0\), then \(d = A\), and the coefficients are \(x = 1\) and \(y = 0\), satisfying \(Ax + By = d\).


    2. Inductive Step: Suppose for \(B\) and \(R = A \mod B\), we have \(Bx_1 + Ry_1 = \text{GCD}(B, R)\). Then, from the Euclidean Algorithm, \(R = A - BQ\), substituting, we get:


    \(\text{GCD}(A, B) = Bx_1 + (A - BQ)y_1 = Ay_1 + B(x_1 - Qy_1)\).


    Thus, the new coefficients \(x = y_1\) and \(y = x_1 - Qy_1\) satisfy \(Ax + By = \text{GCD}(A, B)\). \(\square\)


Now that we have demonstrated the validity of the algorithm, we must demonstrate that the algorithm finishes in a finite number of steps.


  • Proof that the Algorithm Finishes in a Finite Number of Steps:

    Since the Extended Euclidean Algorithm follows the same steps as the Euclidean Algorithm for finding the GCD, it terminates in a finite number of steps as proven for the Euclidean Algorithm. \(\square\)


The Extended Euclidean Algorithm is essential for solving linear Diophantine equations and finding modular inverses, which are vital in cryptographic applications like the RSA encryption algorithm.
